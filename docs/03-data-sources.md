# 데이터 소스 / API / 라이선스

원칙: **브라우저(정적 페이지)에서 직접 쓰려면 — 무료 + CORS 허용 + 상업적 사용 가능**이 이상적.
막힌 곳은 "중간 서버(크롤러)"가 있어야 하고, 법적/약관 위험도 따짐.

## 막힌 곳 (주의)
| 소스 | 문제 |
|---|---|
| Reddit | 비인증 차단·OAuth 필수, 무료=비상업 한정, 상업은 유료 계약(연 $1.2만~), 스크레이핑은 차단+소송 리스크(2025 다수 제소). CORS 미제공 → 서버 필요 |
| 네이버 | 공식 뉴스 API 불명확, 직접 크롤링은 약관/저작권 리스크 |
| 나무위키 | **CC BY-NC-SA(비상업)** → 광고 수익과 양립 불가. 공식 API 없음, 봇 차단, 인기도 피드 없음 |

## 핵심 구분: 학술 데이터
- **메타데이터(제목·초록·인용·관계)** = 유료 논문이든 거의 전부 무료 개방. ← 우주에 필요한 건 이것.
- **본문(PDF)** = 오픈액세스(약 49%)만 무료, 나머지는 페이월(대학 구독).
- 별 클릭 → OA면 본문 무료, 아니면 출판사 링크(+제목·초록·인용은 무료 표시).

## 비영리 vs 상업 메모
- 광고/후원/유료기능/사업홍보/수익화의도 → 상업으로 간주될 수 있음(단순 광고 유무 아님).
- 수익화 계획이면 **상업 가능 소스(위키/HN/OpenAlex)** 로 설계. NC 소스(나무위키)는 비영리 전용.
