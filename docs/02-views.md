# 보여주는 방식 (뷰 모드)

핵심 원칙: **데이터는 한 벌, 뷰는 렌즈.** 같은 별 데이터를 어떻게 좌표로 바꾸느냐가 뷰다.
새 뷰 추가 = 새 프로그램이 아니라 "좌표 변환 함수 + 카메라" 하나 추가.

## 현재 만든 뷰
- **우주 유영** (`space-browser-live.html`): 1인칭 3D 비행(WASD+마우스). 별 클릭→글.
- **밤하늘 데스크톱** (`space-browser-nightsky.html`): 동→서 강 흐름. 스페이스/클릭/터치 누르면 진행.
- **밤하늘 모바일** (`space-browser-nightsky-mobile.html`): 남동→북서 대각선 흐름, 핀치 확대, 위경도 격자.
- **스톤헨지 모바일** (`space-browser-stonehenge-mobile.html`): 3개 스톤(서양/동양/제3세계사), 높이=절대연도, 별자리.

## 뷰가 답하는 질문(성격이 다름)
- 우주 유영 = "이 분야의 지형은?" (공간·관계)
- 스톤헨지 = "시간 축으로 어떻게 흘러왔나?" (역사)
- 밤하늘 = "지금 뭐가 뜨고 지나?" (실시간·관심 동역학)

## 시간 동역학 아이디어 (학술/뉴스에 특히 유효)
재료: 연도별 인기 데이터(예: OpenAlex `counts_by_year`).
- **A. 모멘텀 채널 분리**: 크기=누적, 밝기·맥동=최근 가속. → "오래된 것만 밝다" 편향 해소. (쉬움/임팩트 큼)
- **B. 초신성 플레어**: 급폭발 항목에 번쩍임. 멀리서도 보임.
- **C. 타임라인 재생**: 연도 슬라이더로 분야가 자라는 걸 영상처럼. (헤드라인 기능)
- **D. 프런티어 강조**: 최신·급성장 군집을 색/위치로 → "분야가 어디로 가나".
- **E. 노화/소멸**: 인용 끊긴 옛 항목은 서서히 어두워짐(밤하늘 반감기 재활용).

추천 순서: A+B 먼저 → C → D·E.

## 통합 앱 아이디어
- 런처(`index.html`)에서 뷰 선택 중 → 한 앱에서 모드 전환(공유 데이터 레이어) 목표.
