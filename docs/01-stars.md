# 별 사양 (데이터 → 별 매핑 규칙)

별 하나 = 글/문서/논문 하나. 모든 시각 속성은 데이터에서 계산한다.

## 핵심 개념: weight(무게)
- `weight` = 인기/중요도를 0~1로 정규화한 값. (예: `clamp(sqrt(views/maxViews), 0.12, 1)`)
- 무게가 모든 시각 속성의 기준이 된다.

## 시각 속성

| 속성 | 의미 | 현재 규칙(밤하늘) |
|---|---|---|
| 글로우 반지름 | 인기/중요도 | `30 + weight^2.3 × (화면높이 × 1.25)` (무거우면 화면 초과 가능) |
| 코어(불투명 원) | 밝은 중심 | `글로우 × 0.055` (글로우에 연동, 더 작게) |
| 글로우 밝기(alpha) | 강조도 | `(0.26~) + weight × …` |
| 색 | 분류 | 사이트/게시판/분야별 팔레트 |
| 라벨 크기 | 별 크기 비례 | `12 + weight^2 × 100` |
| 수명/속도 | 체류 시간 | `(10 + weight^2.5 × 220) × rand(0.55~1.45)` — 핫할수록 느리게+편차 |
| 그리는 순서 | 깊이감 | 무거운 별 먼저(뒤) → 작은 별이 앞을 지나감 |

## 별의 종류(상태)
- **일반 별**: 위 규칙대로.
- **초신성**: 임계값 초과(예: 매우 높은 인기/급부상) → 큰 플레어 + 멀리서도 보임 + 라벨 항상.
- **위성(연관 문서)**: 무거운 별 주위를 도는 작은 별. `bind`(결속도)에 따라 궤도 가깝고 오래 머묾. 약하면 멀어지다 먼저 소멸.

## 위성 사양
- 개수: 상위 N개(현재 본체당 최대 5)
- `bind` = 연관 순위 기반 결속도 → 크기·궤도반경·수명에 반영
- 라벨: 상위 1~2개는 희미하게 항상, 나머지는 호버 시

## 데이터 소스별 매핑(같은 별, 다른 의미)
| 소스 | 크기/무게 | 밝기/강조 | 비고 |
|---|---|---|---|
| 위키백과 | 조회수 | (품질등급 가능) | 위성=연관문서 |
| Hacker News | 댓글수/점수 | points | |
| 학술(OpenAlex) | 누적 인용수 | **최근 모멘텀**(연도별 인용 가속) | 위성=참고/유사논문 |
